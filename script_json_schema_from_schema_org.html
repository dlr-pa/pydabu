command line script: json_schema_from_schema_org.py¶

json_schema_from_schema_org.py is a script to define json-ld based on schema.org as a json schema.

usage: json_schema_from_schema_org.py [-h] [-indent i] [-cachefilename f]
                                      [-cachefilepath p]
                                      word [word ...]
Positional arguments:
vocabulary For these words from schema.org the output is generated.
Options:
-indent=[4] In the output the elements will be indented by this number of spaces. default: 4
-cachefilename=['schemaorg-current-https.jsonld.bz2']
 We need data from schema.org. If you set cachefilename to an empty string, nothing is cached. If the file ends with common extension for compression, this comperession is used (e. g.: .gz, .lzma, .xz, .bz2). The file is created in the cachefilepath (see this option). default: “schemaorg-current-https.jsonld.bz2”
-cachefilepath=['/tmp/json_schema_from_schema_org_runner']
 This path is used for the cachefilename. If necessary, this directory will be created (not the directory tree!). default: “/tmp/json_schema_from_schema_org_runner”
